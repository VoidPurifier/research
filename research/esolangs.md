# Esoteric Programming Languages

## 1. Brainfuck (1993)
Extreme minimalism — only 8 commands: `> < + - [ ] . ,`
Turing-complete. Programs look like gibberish.

## 2. Malbolge (1998)
Maximum difficulty — named after 8th circle of Hell.
Code is self-modifying, uses base-3 arithmetic, effect depends on memory address.
Author never wrote a program in his own language. First "Hello World" was brute-forced.

## 3. INTERCAL (1972)
Parody of early programming languages. Uses `PLEASE` keyword (too few = rude, too many = excessively polite).
Operators like `MINGLE`, `UNTWIST`, and three-operand `OR`.
Granddaddy of all esolangs.

## 4. Befunge (1993)
Two-dimensional program flow — code is a 2D grid, instruction pointer moves in four directions.
Programs can look like ASCII art.

## 5. Whitespace (2003)
Only spaces, tabs, and newlines matter. Everything else is ignored.
Created by Edwin Brady and Chris Morris (later created Idris).
Gained fame on Slashdot April 1, 2003 — most thought it was an April Fool's joke.

## 6. Shakespeare Programming Language (2001)
Code as Elizabethan drama — variables are characters, logic is dialogue.
Arithmetic through insults and compliments: "You are as dirty as the sum of a small cat and a large dog."

## 7. Chef (2002)
Code as cooking recipes — variables are ingredients, stacks are mixing bowls.
Design goals include: "recipes should be easy to prepare and delicious."

## 8. Rockstar (2018)
Code as 80s rock ballad lyrics. Created by Dylan Beattie.
Poetic number literals — word length (mod 10) encodes digits.
FizzBuzz reads like actual song about heartbreak.

## 9. Piet (2002)
Code as abstract art — programs are PNG images (Mondrian-style).
Pointer navigates by moving between colored regions. Transitions determine operations.

## 10. Velato (2009, updated 2024)
Code as music — programs are MIDI files.
Commands determined by musical intervals. 2024 update: hands-free mode (whistle to the machine).

## 11. Ook! (2004)
Inspired by Discworld orangutans. Only commands: variations of "Ook" with punctuation.
Brainfuck wrapped in primate communication.

## 12. Schoenberg (2025)
MIDI-based, but programs necessarily sound atonal.
Named after Arnold Schoenberg. Semantics directly match Brainfuck.

## Honorable Mentions
- **LOLCODE** (2007): lolcat syntax (`HAI`, `CAN HAS STDIO?`, `KTHXBYE`)
- **Chicken**: every token is the word "chicken"
- **JSFuck**: valid JavaScript using only `[]()!+` — used in actual XSS attacks
- **Valence** (2025): intentional ambiguity — one program can have two valid interpretations

## Common Themes
1. Minimalism — how few instructions do you need?
2. Difficulty as feature
3. Parody/satire of language design
4. Unusual medium — code as art, music, recipes, poetry
5. Hiding code in plain sight
6. Philosophical questions about what a program is
